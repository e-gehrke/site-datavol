# Critérios de Qualidade

## Propósito

Este documento serve como filtro antes de publicar qualquer conteúdo. Cada pergunta tem um critério objetivo. Se a resposta for "não" em qualquer critério de bloqueio, o post não deve ser publicado.

---

## Critérios de Bloqueio (obrigatórios)

Esses critérios são não-negociáveis. Um "não" aqui bloqueia a publicação.

| # | Critério | Critério de aprovação |
|---|---|---|
| B1 | Os dados numéricos foram verificados na fonte? | Todos os valores têm fonte identificada ou são explicitamente hipotéticos |
| B2 | As fórmulas estão matematicamente corretas? | Conferidas com referência primária |
| B3 | O post faz recomendação individualizada de investimento? | Não faz. O post é educacional. |
| B4 | O post promete retorno financeiro? | Não promete qualquer rentabilidade como certa |
| B5 | O post sugere compra ou venda de ativo específico? | Não sugere. Ativos são usados como exemplos com contexto |
| B6 | O disclaimer está presente em posts sobre produtos de investimento? | Presente no último slide e na legenda |
| B7 | A legislação citada foi verificada como vigente? | Verificada ou identificada como "verificar atualização" |
| B8 | O tom evita linguagem de guru, clichê e promessa? | Ausência de "liberdade financeira", "fique rico", "renda passiva" |

## Critérios de Excelência (diferenciais)

Esses critérios definem a diferença entre um post "aceitável" e um post "excelente".

### Qualidade Técnica

| # | Critério | Indicador |
|---|---|---|
| E1 | O post ensina o mecanismo, não só o resultado? | O leitor entende por que, não só o que |
| E2 | O exemplo numérico usa valores reais e concretos? | Nunca "R$ X mil". Sempre valores específicos |
| E3 | As limitações e premissas estão identificadas? | O conteúdo é honesto sobre o que simplifica |
| E4 | O conteúdo é verificável pelo leitor? | O leitor pode reproduzir o cálculo por conta própria |
| E5 | O post adiciona algo que não está disponível de forma acessível em outro lugar? | O conteúdo justifica sua existência |

### Qualidade Editorial

| # | Critério | Indicador |
|---|---|---|
| E6 | O título é específico e verificável? | Evita "como investir melhor" e usa "como calcular a taxa efetiva do financiamento" |
| E7 | Cada slide tem uma única ideia? | Sem slides com mais de 5 elementos de texto |
| E8 | O CTA é discreto e funcional? | Não vende nada. Indica ação concreta ou salvar |
| E9 | A progressão do carrossel cria curiosidade para continuar? | O leitor tem razão para ir para o próximo slide |
| E10 | O post é útil mesmo para quem não segue a conta? | Conteúdo que existe por si só, não depende de contexto de série |

### Qualidade Visual

| # | Critério | Indicador |
|---|---|---|
| E11 | O design serve o conteúdo, não o contrário? | Nenhum elemento decorativo que distrai |
| E12 | O post é legível em mobile sem zoom? | Testado em tela de 6 polegadas |
| E13 | Os gráficos são diretos e rotulados? | Nenhum gráfico sem título e rótulo de eixo |
| E14 | A hierarquia tipográfica está correta? | Título grande, corpo menor, nota de rodapé menor ainda |
| E15 | As cores seguem o sistema visual do ecossistema? | Nenhuma cor fora da paleta definida em `identidade-visual.md` |

## Critérios por Formato

### Carrossel Educativo (Conta Certa)

| Critério | Padrão |
|---|---|
| Número de slides | 8-12 (máx. 15 em exceções) |
| Primeiro slide tem hook? | Sim, pergunta ou dado surpreendente |
| Exemplo numérico presente? | Obrigatório em posts sobre produtos ou cálculos |
| Progressão lógica? | Cada slide pressupõe o anterior |
| Nível de linguagem adequado? | Leigo com ensino médio completo consegue ler sem dicionário |

### Carrossel Técnico (Nota Técnica)

| Critério | Padrão |
|---|---|
| Número de slides | 10-14 (máx. 16 em exceções) |
| Fórmula com derivação ou intuição? | Pelo menos a intuição, sempre |
| Exemplo numérico presente? | Obrigatório |
| Limitações identificadas? | Pelo menos uma limitação por metodologia |
| Fontes identificadas? | Para resultados específicos atribuídos a autores |

### Cheat Sheet

| Critério | Padrão |
|---|---|
| Funciona impresso em preto e branco? | Sim |
| Máximo de 3 seções? | Sim. Dividir em dois se mais |
| Todas as fórmulas com notação padrão? | Sim |
| Exportado em PDF além do PNG? | Sim, para uso como ferramenta |
| Versão salvávelmente legível em 1080×1080? | Sim |

### Reels

| Critério | Padrão |
|---|---|
| Duração | 45-75 segundos |
| Legível sem som? | Sim, com legendas automáticas ou manuais |
| Número de conceitos no vídeo | Máximo 3 |
| Hook nos primeiros 3 segundos? | Sim |
| CTA verbal no final? | Sim, curto e concreto |

## Escala de Pontuação

Para cada post, somar os critérios de excelência atendidos (E1 a E15). Os critérios de bloqueio são pré-requisito.

| Pontuação | Avaliação | Ação |
|---|---|---|
| 0-7 | Abaixo do padrão | Revisar antes de publicar |
| 8-11 | Adequado | Publicar. Registrar melhorias para próxima versão |
| 12-14 | Bom | Publicar com confiança |
| 15 | Excelente | Candidato a post âncora ou série destacada |

## Monitoramento Pós-Publicação

### Métricas de qualidade (vs. métricas de vaidade)

**Métricas que indicam qualidade:**
- Taxa de salvamento (saves / alcance): ideal >3%
- Compartilhamentos orgânicos
- Comentários técnicos com dúvida real (sinal de conteúdo ativador)
- Taxa de retenção em Reels (>40% chegando ao final)

**Métricas que não indicam qualidade (mas influenciam alcance):**
- Curtidas
- Visualizações (sem contexto de retenção)
- Crescimento de seguidores em um post específico

### Revisão retrospectiva

A cada 30 dias, revisar:
1. Os 3 posts com maior taxa de salvamento
2. Os 3 posts com maior taxa de compartilhamento
3. Padrões de tema e formato que performam melhor
4. Temas que geram mais comentários técnicos

Usar essa análise para ajustar o calendário do próximo ciclo.

## O Que Diferencia Este Ecossistema de um Perfil Genérico

Um post de qualidade neste ecossistema deve passar neste teste:

**Teste do especialista:** Um analista financeiro sênior leria este post e diria "está correto"?

**Teste do leigo:** Uma pessoa sem formação em finanças leria este post e diria "agora entendi"?

Para o Conta Certa: o segundo teste é obrigatório. O primeiro deve ser passado.
Para o Nota Técnica: ambos devem ser passados, mas o primeiro tem peso maior.

Se o post passer nos dois testes, ele merece existir.
